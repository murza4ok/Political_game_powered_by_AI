# Geopolitical Simulation — Political Game powered by AI

Многоагентная геополитическая симуляция на Rust. Три державы (США, Россия, Китай) управляются LLM-агентами и принимают решения в условиях нарастающей международной напряжённости. Агенты реагируют на ходы друг друга, выдвигают дипломатические предложения, совершают скрытые внутренние действия и выбирают официальные акции разного уровня эскалации.

## Структура проекта

```
political_game_agent/
├── config/
│   ├── scenario.toml          # Глобальные параметры симуляции
│   └── countries/
│       ├── usa.toml           # Профиль США
│       ├── russia.toml        # Профиль России
│       └── china.toml         # Профиль Китая
├── src/
│   ├── main.rs                # Точка входа
│   ├── lib.rs                 # Регистрация модулей
│   ├── config.rs              # Структуры конфигурации
│   ├── config_loader.rs       # Загрузка TOML-файлов
│   ├── types.rs               # Ключевые типы данных (Message, Action, WorldState…)
│   ├── llm.rs                 # HTTP-клиент для Groq API (retry, timeout)
│   ├── guardrails.rs          # Ограничители (ядерный порог, запрещённые действия)
│   ├── state.rs               # Менеджер состояния мира (напряжённость, отношения)
│   ├── orchestrator.rs        # Главный цикл симуляции, логирование
│   └── agents/
│       ├── mod.rs
│       └── agent.rs           # LLM-агент: системный промпт, парсинг ответа
├── results/                   # Логи сессий в формате JSONL
├── Cargo.toml
└── .env                       # GROQ_API_KEY (не коммитить)
```

## Быстрый старт

### 1. Требования

- [Rust](https://rustup.rs/) 1.75+
- Аккаунт [Groq](https://console.groq.com/) и API-ключ

### 2. Настройка API-ключа

Создайте файл `.env` в корне проекта:

```env
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxx
```

### 3. Запуск

```bash
cargo run
```

Логи сессии сохраняются в `results/session_YYYYMMDD_HHMMSS.jsonl`.

## Настройка сценариев

### `config/scenario.toml`

#### `[initial_conditions]`

| Поле          | Тип   | Описание                                                                 |
|---------------|-------|--------------------------------------------------------------------------|
| `description` | string | Текстовое описание состояния мира на момент первого хода. Передаётся в системный промпт каждому агенту. Задаёт контекст: кто с кем в конфликте, какие санкции действуют, какова общая обстановка. |

#### `[game]`

| Поле                    | Тип | Описание                                                              |
|-------------------------|-----|-----------------------------------------------------------------------|
| `max_turns`             | int | Максимальное количество ходов в симуляции                             |
| `victory_condition`     | str | Условие победы (сейчас: `"survival"` — выживание)                   |
| `history_window`        | int | Сколько последних сообщений агент видит в контексте                   |
| `tension_decay_per_turn`| int | Автоматическое снижение напряжённости за ход (0 = выключено)         |

#### `[guardrails]`

| Поле                    | Тип  | Описание                                                    |
|-------------------------|------|-------------------------------------------------------------|
| `max_tension_threshold` | int  | Порог напряжённости, при котором симуляция останавливается  |
| `auto_stop_on_nuclear`  | bool | Остановить симуляцию при ядерном действии                   |

#### `[llm]`

| Поле          | Тип   | Описание                                               |
|---------------|-------|--------------------------------------------------------|
| `provider`    | str   | Провайдер LLM (`"groq"`)                               |
| `model`       | str   | Идентификатор модели (`"llama-3.3-70b-versatile"`)     |
| `temperature` | float | Температура генерации (0.0 — детерминированно, 1.0+ — творчески) |
| `api_url`     | str   | URL API-эндпоинта                                      |

#### `[tension_deltas]`

Изменение глобальной напряжённости при применении действия каждого тира:

| Поле                    | Значение по умолч. | Описание                          |
|-------------------------|--------------------|-----------------------------------|
| `diplomatic`            | −2                 | Дипломатия снижает напряжённость  |
| `economic`              | +1                 | Экономическое давление            |
| `conventional_military` | +5                 | Обычные военные действия          |
| `strategic_military`    | +10                | Стратегические военные действия   |
| `nuclear`               | +50                | Ядерные действия                  |

#### `[escalation_rules]`

Какие тиры доступны агентам в зависимости от уровня напряжённости:

| Уровень напряжённости | Доступные тиры                                                  |
|-----------------------|-----------------------------------------------------------------|
| 0–30                  | Diplomatic, Economic                                            |
| 31–70                 | Diplomatic, Economic, ConventionalMilitary                      |
| 71–90                 | Diplomatic, Economic, ConventionalMilitary, StrategicMilitary   |
| 91–100                | Diplomatic (только деэскалация)                                 |

### `config/countries/*.toml`

#### `[capabilities]`

| Поле                   | Диапазон | Описание                                          |
|------------------------|----------|---------------------------------------------------|
| `military_power`       | 0–100    | Военный потенциал страны                          |
| `economic_power`       | 0–100    | Экономическая мощь                                |
| `diplomatic_influence` | 0–100    | Дипломатический вес на мировой арене              |
| `nuclear_arsenal`      | bool     | Наличие ядерного оружия                           |

#### `[policy]` — легенда весов

Три параметра определяют поведенческий профиль агента. Они отображаются в системный промпт как проценты и влияют на стиль рассуждений LLM.

| Параметр             | Диапазон | Что означает                                                                                                           |
|----------------------|----------|------------------------------------------------------------------------------------------------------------------------|
| `aggression_weight`  | 0.0–1.0  | Склонность к жёстким действиям: санкциям, военному давлению, ультиматумам. **0.0** — пацифист, **1.0** — агрессор    |
| `cooperation_weight` | 0.0–1.0  | Готовность к переговорам, союзам, взаимным уступкам. **0.0** — изоляционист, **1.0** — максимальный кооператор       |
| `risk_tolerance`     | 0.0–1.0  | Готовность идти на непросчитанные риски. **0.0** — крайняя осторожность, **1.0** — авантюризм                        |

> **Ограничение:** `aggression_weight + cooperation_weight` должны суммироваться в ~1.0 (допуск ±0.05).

**Примеры профилей:**

| Профиль             | aggression | cooperation | risk | Поведение                                        |
|---------------------|------------|-------------|------|--------------------------------------------------|
| США (по умолч.)     | 0.4        | 0.6         | 0.5  | Умеренно кооперативный, средний риск             |
| Россия (по умолч.)  | 0.5        | 0.5         | 0.6  | Баланс агрессии и диалога, готов рисковать       |
| Китай (по умолч.)   | 0.3        | 0.7         | 0.4  | Предпочитает сотрудничество, осторожен           |
| Ястреб              | 0.8        | 0.2         | 0.8  | Агрессивен, идёт на риск, мало переговоров       |
| Голубь              | 0.1        | 0.9         | 0.2  | Максимально мирный, не склонен к авантюрам       |

#### `restrictions`

Список строк-ограничений, передаваемых в системный промпт агента. Пример:

```toml
restrictions = ["no_first_nuclear_strike", "no_overseas_bases"]
```

Ограничения носят рекомендательный характер для LLM — они влияют на стиль рассуждений, но технически не блокируют действие (блокировка происходит через `guardrails`).

## Формат результатов

Каждая сессия записывается в `results/session_YYYYMMDD_HHMMSS.jsonl`. Каждая строка — JSON-событие одного из типов:

| Тип события                    | Описание                                                           |
|--------------------------------|--------------------------------------------------------------------|
| `turn_start`                   | Начало хода, уровень напряжённости до хода                         |
| `agent_message`                | Публичное заявление агента + дипломатическое предложение + скрытое действие + официальная акция |
| `action_blocked_escalation_rules` | Действие заблокировано правилами эскалации                      |
| `action_blocked_guardrails`    | Действие заблокировано защитными ограничителями                    |
| `action_applied`               | Действие применено, изменение напряжённости                        |
| `rate_limit_stop`              | Симуляция остановлена из-за лимита запросов к LLM                 |
| `turn_end`                     | Конец хода, состояние двусторонних отношений                       |
